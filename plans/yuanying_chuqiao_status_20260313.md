# 元婴出窍状态优化_20260313

## 目标
- 用 `.元婴状态` 优化现有 `.元婴出窍` 调度。
- 当状态回包包含 `状态:元神出窍 归来倒计时:X小时X分钟X秒` 时，解析剩余时间并同步下一次动作时间。
- 当状态回包包含 `状态:窍中温养` 时，判定当前未出窍，可继续执行 `.元婴出窍`。
- 当出窍到期后，通过发送一条任意话语触发系统结算；收到包含 `元神归窍总结` 的回包后，再重新执行 `.元婴出窍`。

## 已识别现状
- 当前 `AutoYuanyingPlugin` 只有固定间隔的 `.元婴出窍` 调度，没有状态查询逻辑。
- 当前遇到“你的元婴正在执行"元神出窍"任务，无法分身”时，会直接忽略，不更新时间。
- 当前项目没有“任意话语触发结算”的专用配置项。

## 已确认需求
- [x] 状态指令固定为 `.元婴状态`
- [x] 触发结算的任意话语固定为 `归来`
- [x] 启动时优先先查一次 `.元婴状态`，不再立刻盲发 `.元婴出窍`
- [x] 收到 `元神归窍总结` 后，立即再次 `.元婴出窍`

## 实现要点（本轮）
- `.元婴出窍` 主循环改为“状态驱动”：优先查 `.元婴状态`。
- 解析 `状态:元神出窍 归来倒计时:X小时X分钟X秒`，同步下一次归来触发时间。
- 解析 `状态:窍中温养`，立即继续 `.元婴出窍`。
- 到达归来时间后，自动发送 `归来` 触发结算；收到 `元神归窍总结` 后立即再次 `.元婴出窍`。
- 遇到“你的元婴正在执行\"元神出窍\"任务，无法分身”时，立即补查 `.元婴状态` 做状态纠偏。

## 执行结果
