# 星宫启阵身份识别修复计划（2026-05-08）

## 目标
- 多身份模式下，某个分身发起 `周天星斗大阵-启` 时，同账号其他身份不能误判为外人邀请并自动 `.助阵`。
- 当前发起启阵的身份仍能识别自己的邀请消息，继续记录邀请消息 id，用于后续 `周天星斗大阵-成` 成功判定。

## 当前问题
- `xinggong.py` 当前只用 `@当前身份名` 判断是否自己的启阵邀请。
- 如果系统邀请文案使用道号、显示名、游戏 ID，或消息被路由到同账号其他身份实例，`my_tag in text` 会失败。
- 失败后代码进入“别人邀请 -> `.助阵`”分支，导致分身启阵时同账号自动跟随助阵。

## 方案
- [x] 在 `AutoXinggongPlugin` 内新增身份匹配辅助逻辑，基于 `Config.active_identity` 识别当前身份。
- [x] 新增同账号身份匹配逻辑，基于 `Config.identities` 的身份字段识别“自家任意身份”。
- [x] `周天星斗大阵-启`：
  - [x] 命中当前身份：按自己的邀请处理，记录 `_qizhen_last_invite_msg_id` / `_qizhen_last_invite_slot`。
  - [x] 命中同账号其他身份：直接忽略，不发送 `.助阵`，也不污染当前身份的启阵状态。
  - [x] 未命中任何自家身份：保持原行为，必要时 `.助阵`。
- [x] `周天星斗大阵-成`：成功归属判断从 `@当前身份名` 扩展为当前身份 token 匹配，保留 invite edit id 兜底。

## 验收
- [x] 新增星宫插件单测：当前身份为分身，邀请文案只出现分身道号/游戏 ID 时，不发送 `.助阵`，并记录自己的 invite id。
- [x] 新增星宫插件单测：当前身份为主魂，邀请文案命中同账号分身时，不发送 `.助阵`，且不记录主魂 invite id。
- [x] 新增星宫插件单测：外人邀请仍会返回 `.助阵`。
- [x] 运行 `./venv/bin/python -m unittest tests.test_xinggong tests.test_multi_account`。

## 待确认
- 推荐策略：同一个账号下配置的所有身份都视为“自家身份”，因此互相不会自动助阵。
