# 观星事件专用路由计划（2026-04-27）

## 目标
- 让“其他人的观星系统回应”在多身份模式下能被配置了观星劫持的化身处理。
- 只改变观星事件消息的分发路径，避免把所有公共消息广播给所有身份。

## 当前问题
- `runtime.py` 目前每条消息只选一个身份处理。
- 其他人的 `星盘显化` 通常不包含我方化身身份标识，也不是回复化身消息。
- 因此这类消息会落到当前激活身份或文本命中的其他身份，化身的 `xinggong` 插件可能完全收不到。

## 方案
- [x] 在 `runtime.py` 增加观星事件候选判断，仅识别 `星盘显化` / `天机异动` 这类观星劫持源消息。
- [x] 把 `enable_xinggong_guanxing` 加入身份覆盖项，允许主魂关闭、指定化身开启。
- [x] 只选择一个观星监听身份：按账号身份配置顺序，取第一个实际 `enable_xinggong=True` 且 `enable_xinggong_guanxing=True` 的身份。
- [x] 观星候选消息若存在监听身份，则直接路由给该监听身份；不存在监听身份时保持原默认路由。
- [x] 执行动作时沿用监听身份的 `_execute_action(..., identity_key=...)`，保证由化身执行 `.观星` / `.改换星移`。
- [x] 不广播 `观星台`、`周天星斗大阵`、日常回包等其他公共消息。

## 验收
- [x] 新增测试：当前激活身份为主魂时，其他人的 `星盘显化` 命中化身配置事件后，唯一开启观星劫持的化身能注册观星劫持窗口。
- [x] 新增测试：多个身份开启观星劫持时，只按身份配置顺序选择第一个监听者，不重复抢同一事件。
- [x] 新增测试：普通公共消息不会广播给化身，保持原路由行为。
- [x] 运行相关测试：`./venv/bin/python -m unittest tests.test_multi_account tests.test_xinggong`。

## 已确认
- 观星事件只允许一个身份监听。
