# TG 主魂/化身多身份切换计划（2026-04-21）

## 目标
- 在单个 TG session 下支持主魂 + 多个化身身份切换。
- 当前激活身份持久化，重启后恢复。
- 每个身份拥有独立插件配置与独立状态存储。
- 未激活身份任务到期时，自动切换后执行，失败则保留队列重试。
- 网页支持配置身份组、查看当前激活身份与切换参数。

## 已确认规则
- 切换命令模板可配置，系统回包自动识别切换结果。
- 主魂与化身同级可切换。
- 化身列表来源以网页配置为准。
- 主魂用 TG 用户名 + 道号识别，化身用道号 + 游戏 ID/切换参数识别。
- 多身份任务冲突按到期时间先后执行。

## 执行清单
- [x] 扩展配置模型，加入身份组、激活身份与切换规则
- [x] 实现身份状态命名空间与激活身份持久化
- [x] 改造 AccountRunner，接入身份编排与自动切换执行
- [x] 扩展网页编辑页与仪表盘展示身份组
- [x] 增加针对性测试并验证兼容旧单身份配置

## 实施结果
- 新增 `IdentityProfile` 与身份组配置，旧单身份配置会自动回填 `main` 身份。
- 新增身份切换协调器，记录当前激活身份并识别切换成功/失败回包。
- 运行时按身份创建独立插件实例与独立状态命名空间，共享同一个 TG session。
- 定时任务发送前会确保目标身份已激活，未激活时先发送切换命令。
- 网页账号表单新增身份组 JSON 与切换规则配置，仪表盘显示持久化激活身份。
