# 星宫观星劫持功能计划（2026-03-21）

## 目标
- 在现有 `xinggong` 插件中新增“观星事件监听 + 临近整点劫持 + 改换星移转移”能力。
- 监听群话题内所有相关系统消息；若出现高价值观星结果，则在结算前为本人执行抢占流程。
- 尽量复用现有 `xinggong.py` 的定时与消息匹配模式，避免新增独立大型状态机。

## 已确认需求
- 监听关键词：系统回复包含 `【星盘显化】`，并关注其中的“天道演化”结果。
- 高价值结果默认包含：`地磁暴动`、`星辰异象`；并且这些事件名需要做成可配置列表。
- 刷新节奏：北京时间每 3 小时一个周期（00:00 / 03:00 / 06:00 ...）。
- 有效期：每次观星结果 5 分钟内可操作。
- 结算策略调整为：不是命中后立即 `.观星`，而是在结算前 5 分钟窗口内再发；当前指定为“结算前 3 分钟发送 `.观星`”。
- 改换星移仍在整点前 1 秒卡点发送；该“改换星移提前量”需要可配置。
- `.观星` 提前量也应做成配置项（当前需求为 180 秒）。
- 范围：监测系统对所有人的相关消息，而不是只看自己的回包。
- 预观星命令：`.观星`。
- 转移命令：`.改换星移 @新目标`。
- 当前目标 TG username：`salt9527`；目标 username 需要做成配置项。
- 同一 3 小时窗口内只会出现一种事件，且同一 3 小时窗口只需监听一次事件，无需做多事件竞争策略。
- 需要避免系统延迟误判：若有人在旧窗口结算前发 `.观星`，其 `【星盘显化】` 回包可能在新窗口开始后才到达；这种迟到消息不能当作新窗口事件。
- `【星盘显化】` 的消息格式已确认：包含“下一次天道演化将是：【Good·...】/【Bad·...】”以及“当前天命所归：@某人”。
- 我方 `.观星` 的即时回包已确认：`你开始消耗xx点修为，尝试扭转因果...(成功率xx%)`。
- `.观星` 失败文案已确认：`你今日已观星一次，天机不可多泄，请明日再来`。
- 整点结算消息是独立新消息，不 reply 任何人，例如 `【天机阁快报·星辰异象】`。

## 代码现状
- `app.py` 已同时监听 `NewMessage` 和 `MessageEdited`，并会把话题内消息派发给 `xinggong` 插件。
- `xinggong.py` 已有成熟的“定时 loop + 消息回包驱动 + reply_to_msg_id/消息编辑匹配”范式，可作为新增功能宿主。
- 当前 `xinggong` 插件尚未实现 `.观星` / `.改换星移` 相关状态与调度。

## 初步方案
- 直接扩展 `xiuxian_bot/plugins/xinggong.py`，不单独拆新插件。
- 新增“观星机会”状态：记录命中的系统消息 id、命中时间、对应结算整点、是否已安排预观星/改换星移。
- 当收到目标关键词消息时：
  - 解析是否为高价值结果；若不是则忽略。
  - 若是，则根据下一次整点安排两个动作：
    1. 结算前预发送 `.观星`
    2. 结算前更晚一点回复目标消息发送转移指令
- 需要处理同一窗口内多条候选消息的覆盖/去重，以及消息编辑场景。

## 2026-03-26 补充调整（待执行）

### 新问题
- 现网日志确认：高价值事件经常先以 `【天机异动】...【Good - 星辰异象】` 形式出现，而当前实现只把 `【星盘显化】` 作为观星事件源，导致未注册结算窗口，进而不会在结算前 180 秒自动发送 `.观星`。
- 现网消息部分来自图片文字提取，存在空格、换行、全角/半角、标点（如 `-` / `·` / `：`）漂移风险，当前直接 `in` 匹配较脆弱。
- 用户已确认：系统当前仍维持“回复 `.观星` 指令消息”的旧逻辑，因此“我自己的 `【星盘显化】` 回包”仍可优先通过 reply 相关 id 识别。

### 调整后的方案
- 新增一层**全局可复用的文本归一化能力**，供所有插件的“文本匹配判定”调用，但**不直接覆盖原始 `ctx.text`**，避免污染日志、原文解析与 reply 相关行为。
- 归一化目标：
  - Unicode NFKC 规范化
  - 去除零宽字符 / 不可见字符
  - 统一空白（压缩或按需移除）
  - 弱化括号、冒号、连接符、圆点等符号差异
  - 尽量兼容 OCR 带来的全角/半角、换行、空格、标点漂移
- 推荐落点：抽成共享文本工具模块，由各插件在需要“鲁棒匹配”时显式调用；不在 `runtime.py` / `tg_adapter.py` 层全局改写消息正文。
- 观星事件检测从“只认 `星盘显化`”扩展为“归一化后识别多锚点组合”：
  - `星盘显化`
  - `天机异动`
  - `下一次天道演化`
  - `当前天命所归`
  - 并结合高价值事件配置项（默认 `星辰异象`、`地磁暴动`）共同判定
- 保留现有 reply/id 识别链路作为强信号；归一化文本匹配主要用于：
  - 外部高价值事件注册
  - OCR/标点变体兼容
  - 其他插件未来的鲁棒文本识别复用
