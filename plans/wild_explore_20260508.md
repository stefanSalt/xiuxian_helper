# 野外历练功能计划 - 2026-05-08

## 目标
- 新增“野外历练”自动化功能。
- 开启后发送野外历练指令，基础指令为：`.野外历练`。
- 支持配置历练策略：谨慎、均衡、深入。
- 固定每 2 小时触发一次。
- 每轮触发时重复发送一次同样指令，降低命令丢失风险。

## 默认行为
- 功能默认关闭，通过账号配置开启。
- 支持身份级覆盖，避免多身份场景下所有身份同时执行。
- 默认策略为“深入”，实际发送指令为：`.野外历练 深入`。
- 策略可在 Web 配置中修改：
  - 谨慎：发送 `.野外历练 谨慎`
  - 均衡：发送 `.野外历练 均衡`
  - 深入：发送 `.野外历练 深入`
- 每轮发送两次：第一次立即发送，第二次延迟 10 秒发送。
- 第二次补发延迟可在 Web 配置中修改。
- 下一轮从第一次发送后间隔 7200 秒调度。

## 验证方式
- 新增插件单元测试验证：
  - 启用后 bootstrap 会调度首次任务。
  - 执行一次任务会按配置策略发送野外历练指令两次。
  - 第二次发送使用独立延迟，下一轮按 7200 秒调度。
  - 冷却回包会解析剩余时间，并按到期时间重发。
  - 正常回包会按配置间隔调度下一轮。
- 配置测试验证：
  - 策略仅允许：谨慎、均衡、深入。
  - 补发延迟从 Web 表单写入后能被插件读取。
- 运行配置/Web 相关测试，确认新字段不会破坏现有表单解析。
