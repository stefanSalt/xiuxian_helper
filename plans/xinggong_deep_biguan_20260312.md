# 星宫启阵 × 深度闭关联动_20260312

## 目标
- 在现有星宫 `.启阵` 定时逻辑之上，增加“深度闭关联动”模式。
- 当启阵成功后，自动执行一次“强行出关 -> 再次深度闭关”，以便让后续闭关吃到阵法收益。
- 在该次启阵成功后的第 5 小时，再执行一次“强行出关 -> 再次深度闭关”。
- 该流程持续到下一次启阵成功后重新开始新一轮。
- 增加独立开关；关闭时保持现有行为：只做定期启阵，不做任何深度闭关联动。

## 已识别的实现边界
- 现有 `AutoXinggongPlugin` 负责：观星台、启阵、助阵、每日问安。
- 现有 `AutoBiguanPlugin` 负责普通 `.闭关修炼` 的冷却解析与重试。
- 当前仓库里尚未发现“深度闭关/强行出关”的现成实现或命令常量。
- 若直接新增联动，必须先明确它与当前普通闭关插件是否并行，否则容易互相打架。

## 已确认需求
- [x] 深度闭关指令：`.深度闭关`
- [x] 强行出关指令：`.强行出关`
- [x] 闭关状态检查指令：`.查看闭关`
- [x] 启阵成功后，若当前不在深度闭关，也要直接补发深度闭关
- [x] 启用该模式后，现有自动闭关（`.闭关修炼`）需要自动暂停，避免并发冲突
- [x] 该功能需要独立开关；关闭时仅保留原有定期启阵

## 实现要点（本轮）
- 已知 `.查看闭关` 的关键回包：
  - 未处于深度闭关：`你并未处于深度闭关之中`
  - 处于深度闭关：`你正在深度闭关，预计还需 X小时X分钟X秒 即可功成圆满。`
- 在星宫插件内增加“深度闭关联动”状态机：启阵成功 -> 查状态 -> 必要时强行出关 -> 进入深度闭关。
- 在每次启阵成功后额外安排一个“+5小时刷新”任务：查状态 -> 强行出关/重新进入深度闭关。
- 下一次启阵成功时，取消上一轮未完成的联动任务，并开启新一轮。
- 普通闭关插件在“星宫启阵×深度闭关”开关开启时自动暂停。
- 优先做最小范围、无持久化实现；重启后的联动状态从下一次启阵成功重新建立。
