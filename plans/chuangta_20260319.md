# 每日闯塔功能计划（2026-03-19）

## 目标
- 为当前自动化系统加入 `.闯塔` 每日任务。
- 与现有元婴插件协同：尽量在“元婴未出窍/窍中温养”时执行，以放大利益。
- 兼容未到元婴期/没有元婴功能的账号，不因缺少元婴状态而卡死。

## 非目标
- 不重构现有调度框架。
- 不顺带扩展其他日常玩法。
- 在没有稳定回包样例前，不做复杂奖励统计。

## 已确认接入点
- `xiuxian_bot/plugins/yuanying.py`：已有 `.元婴状态` / `.元婴出窍` / `归来` 状态机，可复用“窍中温养/出窍中/归窍总结”判断。
- `xiuxian_bot/app.py`：应用入口统一注册插件与 bootstrap。
- `xiuxian_bot/config.py`：已有插件开关与间隔配置模式，可按同样方式新增闯塔配置。
- `tests/test_yuanying.py`：已有元婴行为测试，后续可补闯塔协同测试。

## 方案草案
- 新增独立插件更合适：如 `xiuxian_bot/plugins/chuangta.py`。
- 该插件只关心“今天是否已完成闯塔、是否待执行、是否需要等待元婴归来”。
- 若检测到元婴插件可用：
  - `状态:窍中温养` -> 立即 `.闯塔`
  - `状态:元神出窍` -> 延后到归来后再闯塔
  - 收到 `元神归窍总结` 后，若当天尚未闯塔，则补发 `.闯塔`
- 若元婴插件未启用/未达元婴期：按普通每日任务执行 `.闯塔`，不依赖 `.元婴状态`。
- 启动恢复：若当天尚未确认完成，可在启动时做一次补检查/补执行。

## 已确认需求
- [x] 每天闯塔次数：固定 1 次/天。
- [x] 固定时间：每日 14:15。
- [x] 触发策略：固定时间启动；若元婴未出窍则立即执行，若已出窍则等归来后补做。
- [x] 元婴出窍协同：元婴出窍 8 小时后必定可归来，闯塔插件只需等待现有元婴插件完成结算后补做。
- [x] 未到元婴期时：直接照常 `.闯塔`，不依赖元婴状态。
- [x] 今日已做识别：系统回包包含“你今日已挑战失败”即视为今日任务完成；闯塔本身不要求通关成功。
- [x] 深度闭关协同：深度闭关不影响 `.闯塔`，无需为此暂停或改道。

## 拟落地逻辑
- 新增独立插件 `chuangta`，单独维护“今日是否已闯塔完成/是否等待补做”。
- 每日 14:15 触发一次主流程：
  - 若当天已标记完成，则跳过。
  - 若元婴插件启用，则先发 `.元婴状态` 判定是否 `窍中温养` / `元神出窍`。
  - 若未启用元婴插件，则直接发 `.闯塔`。
- 当收到 `.元婴状态` 回包：
  - `状态:窍中温养` -> 立即发 `.闯塔`
  - `状态:元神出窍` -> 记录“等待归来后补做”，不立即发塔
- 当后续收到 `元神归窍总结`：
  - 如果当天尚未闯塔且处于“等待归来后补做”，立即发 `.闯塔`
- 当收到闯塔结果回包：
  - 若包含“你今日已挑战失败” -> 标记今日已完成
  - 其他与闯塔相关的回包，默认也标记今日已完成，避免同日重复发送
- 重启恢复：
  - 若当天未完成且已过 14:15，则启动后立即补跑一次主流程
  - 若重启时元婴仍在出窍，则继续等待其 `元神归窍总结` 再补闯塔
