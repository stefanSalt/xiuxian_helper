# 凌霄宫插件扩展计划（引九天罡风 + 问心时机）（2026-04-12）

## 目标
- 为 `xiuxian_bot/plugins/lingxiaogong.py` 增加 `.引九天罡风` 自动施放能力。
- 将 `.问心台` 的默认触发时机从“当日首次可登阶前”改为“当日第 N 次有效登天阶后”。
- 在网页配置中新增“第几次登天阶后施放问心台”设置，并保持旧账号配置兼容。

## 当前代码现状
- 现有凌霄宫插件只自动处理：
  - `.天阶状态`
  - `.问心台`
  - `.登天阶`
- 当前策略：
  - 启动先查 `.天阶状态`
  - 若当天未问心，则优先 `.问心台`
  - 冷却结束后自动 `.登天阶`
- `Config` / `web.py` 当前只有：
  - `enable_lingxiaogong`
  - `enable_lingxiaogong_wenxintai`
  - `enable_lingxiaogong_dengtianjie`
  - `lingxiaogong_poll_interval_seconds`

## 本次建议改动
1. `Config`
   - 新增 `enable_lingxiaogong_jiutian`
   - 新增 `lingxiaogong_wenxintai_after_climb_count`

2. `web.py`
   - 凌霄宫配置区新增：
     - 自动引九天罡风开关
     - 问心台在第几次登天阶后施放

3. `lingxiaogong.py`
   - 增加 `.引九天罡风` 状态持久化与调度
   - 增加“当日有效登天阶次数”计数
   - 将问心触发条件改为：
     - 当日未问心
     - 当日有效登天阶次数已达到配置阈值
   - 保持冷却驱动式登天阶逻辑不变

4. `tests`
   - 增加配置解析测试
   - 增加网页表单测试
   - 增加凌霄宫插件调度测试
   - 增加状态持久化恢复测试（如涉及新状态字段）

## 已确认口径
1. “第几次登天阶之后施放”：
   - 已确认：**作用对象是 `.问心台`**
   - 即：当天先正常登天阶，达到第 N 次有效登天阶后，再去施放 `.问心台`

2. “有效登天阶次数”：
   - 已确认：只统计**真正进入结算的登天阶回包**
   - 不统计“冷却中，请稍后再试”这类拒绝回包

3. `.引九天罡风` 调度：
   - 已确认：解锁后启用
   - 已确认：成功施放后按 **12 小时冷却** 记录并持久化
   - 已确认：冷却到点后直接施放，不依赖“达到第 N 次登天阶”
