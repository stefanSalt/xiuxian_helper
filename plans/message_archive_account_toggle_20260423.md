# 消息归档账号级开关计划（2026-04-23）

## 目标
- 给消息归档增加账号级开关。
- 每个 TG 账号可在网页配置中单独开启 / 关闭消息归档。
- 开关只跟随账号配置，不区分主魂与化身。
- 默认保持开启，避免升级后现有账号突然停止归档。

## 方案
- [x] 在 `Config` 增加账号级字段 `enable_message_archive`，默认 `True`。
- [x] 在网页账号表单增加“消息归档”开关，并纳入表单解析。
- [x] 在运行时归档入口按 `base_config.enable_message_archive` 判断是否写入归档。
- [x] 补测试覆盖：默认开启、关闭后不归档、网页表单可保存开关。
- [x] 跑消息归档与多账号相关测试。

## 边界
- 不调整全局清理策略：`MESSAGE_ARCHIVE_*` 仍然是系统级配置。
- 不按主魂 / 化身拆分开关。
- 不删除已有归档数据；关闭后只是不再新增。
