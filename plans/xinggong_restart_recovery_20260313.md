# 星宫启阵重启恢复_20260313

## 目标
- 在脚本重启后，自动恢复“本轮启阵 × 深度闭关”联动，不再因为进程重启而丢失本轮第 5 小时刷新。
- 采用“重发 `.启阵` -> 读取冷却 -> 倒推出本轮启阵成功时间”的方案，尽量避免新增持久化状态。

## 已确认需求
- [x] 重启后可通过再次发送 `.启阵` 并解析“请在 X小时X分钟X秒 后再次启阵”来倒推出本轮启阵成功时间。
- [x] 倒推出本轮启阵成功时间后，需要恢复本轮第 5 小时的深度闭关联动。
- [x] 恢复时必须先检查 `.查看闭关`。
- [x] 若当前未处于深度闭关：立即发送 `.深度闭关`。
- [x] 之后仍以“推导出的启阵成功时间 + 5 小时”为基准，执行一次 `.强行出关` -> `.深度闭关`。

## 待确认边界

- [x] 若重启后第一次 `.启阵` 收到的是“你已发布启阵邀请，请勿重复操作”，本轮先维持最小修改：仅等待阵成/过期，不做额外探测。
